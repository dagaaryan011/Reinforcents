# 1. Market Maker using SAC:
The agent constantly collects the parameters of all orderbooks like: highest bid, lowest ask, their option greeks etc.<br>
All orderbook parameters pass through a selector network which selects one option. <br>
The agent places both buy and sell orders in that option based on the state of that option. <br>
The selector tries to select the option with maximum volume. <br>
The orders are placed using 4 action outputs of actor network for buy price, sell price, buy amount, sell amount. The actor network along with 2 critics and 2 value networks are trained using SAC algorithm. <br>


# 2. Institutional Trader using DDPG:
The institutional agent, operating on the DDPG algorithm, constructs a comprehensive state vector for its decision-making process. This state includes:

Its current portfolio status, detailing cash balance and existing option holdings.

Market data for a select number of observed options.

Contextual market sentiment derived from a long-term RNN model.

This complete state is then fed into the Actor network, which deterministically outputs a continuous action vector. Each value in this vector corresponds to the desired trade strength for one of the observed options, and the agent executes a trade on the single option with the highest absolute action value.

The learning process is driven by the Critic network, which evaluates the quality of the action taken in a given state by estimating the expected future reward. The algorithm updates both the Actor and Critic networks through the following steps:

The Critic is trained by comparing its Q-value prediction to a more stable target value generated by the target networks.

The Actor is then updated using the policy gradient provided by the Critic, which guides it to produce actions that lead to higher rewards.

Experiences, consisting of states, actions, rewards, and next states, are stored in a replay buffer and sampled randomly to train the networks, ensuring stable learning.
